| Field | Description | Simulation Role |
|-------|-------------|------------------|
| `identity_pubkey` | Validator ID | Node uniqueness |
| `average_stake_percentage` | Stake weight | Voting power |
| `average_skip_rate` | Slot reliability | Liveness simulation |
| `average_mev_to_validator` | MEV earnings | Reward modeling |
| `average_median_vote_latency` | Vote arrival timing | Fork race modeling |
| `average_votes_cast` | Vote activity | Finality quorum input |
| `asn`, `country`, `region` | Network geography | Latency simulation |

## üß± Simulation Architecture

This proposal is language-agnostic. Suggested modules:

- `ValidatorSet`: Ingest and normalize JSON data
- `RelayNetwork`: Assign latency/failure risks
- `ConsensusRound`: Simulate fast vs. slow finality
- `RewardEngine`: Calculate rewards, MEV share, penalties
